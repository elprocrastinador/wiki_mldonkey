Описание

Этот список содержит IP адреса компьютеров которым разрешено управлять MLdonkey клиентом (т.е. подключаться через порты интерфейсов). По умолчанию при первой установке разрешён доступ только с адреса локальной машины \localhost\ (ip адрес 127.0.0.1).

allowed_ips является не скалярной опцией в ней поддерживается список опций(значений\values). Для полного понимания того, как устанавливать множественные (несколько\multiple) значения опций, смотрите EnteringMultipleValues.

Помимо отдельных IP адресов, также вы можете указать определённый диапазон сетей. Тем не менее дважды подумайте перед тем как делать так, это серьёзная угроза безопасности.

Если вы предпочитаете иметь доступ к ядру через процесс аутефикации(проверки имя-пароль), шифрование трафика и как опцию сжатие передаваемых данных, настройте [[SshTunnel| Ssh туннелирование]

Значение по умолчанию

Допустимым IP адресом по умолчанию является 127.0.0.1.

Размещение

Эта опция находиться в основном файле конфигурации downloads.ini

Примеры

Командой в консоли телнета или вэб-интерфейса: 

> set allowed_ips "127.0.0.1 192.168.15.255"

будут установлены допустимые(разрешённые) ip адреса подключающихся компьютеров.

В формате рассчитанном для записи в файл downloads.ini это будет выглядеть так:
["127.0.0.1"; "192.168.15.255"]
. Такие настройки позволят подключаться через порт интерфейса с локальной машины (адрес 127.0.0.1) и подсети 192.168.15.0/24 (это всё множество компьютеров начинающихся с 192.168.15. в вашей локальной сети.

Если вы захотите добавить весь интернет. ВНИМАНИЕ: это потенциально опасно, мы предупредили! То вам необходимо указать следующий промежуток командой в консоли: 

 > set allowed_ips "0.0.0.0-255.255.255.255"

Также его можно прописать в разделе Allowed IPs файла downloads.ini

Обозначения CIDR (Бесклассовая адресация)

Начиная с версии 2.8.0 поддерживает другой синтаксис (перевод правилен ?) 

5264: allowed_ips: Поддерживает CIDR формат адреса/маски и IP диапазона (pango)
* старый синтаксис преобразуется в формат CIDR:
  - 192.168.000.255 -> 192.168.000.000/24
  - 192.168.255.255 -> 192.168.000.000/16
  - 192.255.255.255 -> 192.000.000.000/8
  - 255.255.255.255 -> 000.000.000.000/0
* для подсчёта по новому синтаксису используйте онлайн калькулятор: http://www.subnet-calculator.com/cidr.php для
  - CIDR, на подобии 192.168.0.106/30 (192.168.0.104 - 192.168.0.107)
  - IP диапазонов, подобных 192.168.1.123 - 192.168.1.133

Когда вы ставите новую версию, файл конвертируется автоматически под новые настройки, это не работает если вы возвращаетесь с более новой на старую версию, старые версии не понимают нового синтаксиса и вам придётся настраивать файл разрешённых IP адресов ( allowed_ips ) снова.
