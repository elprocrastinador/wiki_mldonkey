Конфигурационные .ini файлы MLDonkey



При первом запуске mldonkey создает довольно много новых файлов. На этой странице объяснено для чего нужны эти файлы.

Для каждой сети которая включена и к которой может подключиться mldonkey создаются .ini файлы для хранения переменных:

audiogal.ini

содержит параметры для пиринговой сети audiogalaxy. MLDonkey больше не поддерживает эту сеть. Поэтому он точно не будет создан.

directconnect.ini

содержит параметры для пиринговой сети ДЦ++ (Direct Connect \dc++\ )

downloads.ini

содержит основные переменные mldonkey, которые могут влиять на работу более чем в одной сети, например и edonkey2000(e2k сеть) и сети overnet. Это главный файл конфигурации.

file_sources.ini

содержит источники к которым вы можете подключиться

Files.ini_RU

содержит md4-хэш md4-hash всех файлов которые сейчас скачиваются

friends.ini

содержит IP адреса и имена друзей

limewire.ini

содержит параметры для сети limewire так же известной как gnutella

openft.ini

содержит параметры для сети openFT (главный протокол для демона giFT)

opennap.ini

содержит параметры для пиринговой сети Open Napster

searches.ini_RU

содержит созданные вами поисковые запросы

server.ini

содержит параметры edonkey-server который встроен в mldonkey.

servers.ini

содержит список серверов к которым вы можете подключиться

shared_files.ini

содержит имена и хэш md4-hash файлов которые вы расшарили (пердоставили в общий доступ).

soulseek.ini

содержит параметры для сети Soulseek

Все эти файлы в текстовом формате. Когда используется файл из другой версии MLDonkey, недостающие параметры будут установлены в их стандартные (по умолчанию) значения, нераспознанные параметры будут стёрты (удалены). Как правило, это безопасно, так что вы можете переходить со дной версии MLDonkey на другую просто заменив исполняемые файлы (файлы ядра, графического интерфейса пользователя и т.п.) -- так же это справедливо для списков графического интерфейса (gui_donkey_options_panel, gui_overnet_options_panel,...).


ЗАМЕЧАНИЕ: Так как при работе ядра в эти файлы периодически перезаписываются обновленные данные, вам необходимо остановить ядро программы перед их редактированием. Почти все настройки могут быть изменены на лету используя команду set в MLDonkey. (Прим. перев.: так же это касается и вэб-интерфейса)
