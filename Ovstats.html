Syntax

~> ovstats

Description

(obsolete) Shows a list of known Overnet peers and current search information. All peers are identified by an md4 and the peer with the md4 closest to a particular file's md4 is responsible for tracking available sources for that file.

For all files that you are trying to download, MLdonkey tries to find peers with md4 close to to those files. For example, in the sample output you can see that my search for file \"4AC7A501E7569A8F7F6C4A5BA49391D4\" has resulted in an increased number of peers with md4s starting with \"4A\" in the matrix, since these are likely to give me sources for that file.

Each file is identified by an md4. You are responsible for keeping track of the sources for the file with the md4 that closest matches you client md4 (see ovmd4 to find out your clinet md4), but all files consist of a set of chunks, each with their own md4.

In the matrix you can also see increased counts for client md4s that you're not trying to download from. These I beleive correspond to the the md4 of sources for the files that you are responsible for keping track of. In other words, these are source md4s that you report back to peers searching for files with and md4 similar to your clients md4.

Exactly why there's always at least 8 peers, I'm not sure.

This is as far as I've understood it, it's probably more complicated than this... /Kokamomi
